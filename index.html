    function useLocalStorage(key, initialValue) {
        const [storedValue, setStoredValue] = useState(() => {

            } catch (error) {
                console.error(error);
                return initialValue;

            try {
                const valueToStore = value instanceof Function ? value(storedValue) : value;

                window.localStorage.setItem(key, JSON.stringify(valueToStore));

    const resizeImage = (file, maxWidth, maxHeight) => {
        return new Promise((resolve, reject) => {
            const reader = new FileReader();
            reader.onload = (e) => {
                const img = new Image();
                img.onload = () => {
                    const canvas = document.createElement('canvas');
                    let width = img.width;
                    let height = img.height;

                    if (width > height) {
                        if (width > maxWidth) {
                            height *= maxWidth / width;
                            width = maxWidth;
                        }
                    } else {
                        if (height > maxHeight) {
                            width *= maxHeight / height;
                            height = maxHeight;
                        }
                    }

                    canvas.width = width;
                    canvas.height = height;
                    const ctx = canvas.getContext('2d');
                    ctx.drawImage(img, 0, 0, width, height);
                    resolve(canvas.toDataURL('image/jpeg', 0.8));
                };

La Spiegazione Semplice

Il motivo per cui i dati non "seguono" l'app quando la pubblichi è che localStorage salva le informazioni nel browser, ma le lega all'indirizzo specifico del sito web.

  1. Quando lavori sul tuo computer, l'indirizzo è probabilmente qualcosa come file:///C:/Users/tuo-nome/.../index.html. I dati vengono salvati per questo "sito".

  2. Quando pubblichi su GitHub Pages, l'indirizzo diventa https://tuo-username.github.io/nome-repo/. Questo è un sito web completamente diverso per il browser.

Di conseguenza, il browser mantiene i dati separati. Quelli salvati localmente rimangono sul tuo computer, e quelli che inserirai sul sito di GitHub Pages verranno salvati solo per quell'indirizzo.

La Soluzione: Importa ed Esporta i Tuoi Dati

Per risolvere questo problema e darti il pieno controllo sui tuoi dati (permettendoti di fare backup o spostarli tra computer), ho aggiunto una funzionalità di Import/Export.

Nella parte superiore dell'app, accanto al titolo, ora troverai due nuovi pulsanti:

  • Esporta Dati: Cliccando qui, l'app genererà un singolo file (gestione_backup.json) contenente tutti i tuoi ordini, spese e scorte. Puoi salvare questo file sul tuo computer.

Perché i Dati Non si Sincronizzano Automaticamente

Immagina che i dati salvati dal browser (localStorage) siano come oggetti chiusi in una scatola.

  1. La tua "scatola" locale: Quando usi l'app sul tuo computer, il browser mette i dati in una scatola etichettata con l'indirizzo del file, ad esempio: file:///C:/.../index.html.

  2. La "scatola" di GitHub Pages: Quando visiti l'app pubblicata, l'indirizzo è https://tuo-utente.github.io/tua-repo/. Il browser usa una scatola completamente diversa, con un'etichetta diversa.

Il browser, per motivi di sicurezza, non permette a queste due scatole di comunicare tra loro. Non c'è modo per il sito su GitHub Pages di "sbirciare" nella scatola del tuo computer locale e prenderne i dati automaticamente.

GitHub Pages è un servizio per ospitare siti "statici", ovvero senza un database sul server. L'unico posto dove l'app può salvare dati è il browser dell'utente, e quel salvataggio è sempre legato all'indirizzo del sito che si sta visitando.

La Soluzione Migliore e Più Sicura: Import/Export

La soluzione che ti ho fornito con i pulsanti Importa/Esporta è lo standard del settore per questo tipo di app offline. Non è un "workaround", ma una funzionalità che ti dà il pieno controllo e la sicurezza dei tuoi dati.

Vantaggi di questo approccio:

  • Backup: Puoi salvare i tuoi dati al sicuro sul tuo computer.

  • Privacy: I tuoi dati rimangono sempre sotto il tuo controllo, non vengono mai inviati a un server esterno.

  • Portabilità: Puoi spostare i tuoi dati tra diversi computer o browser semplicemente importando il file di backup.

Workflow per GitHub Pages

Il processo corretto per gestire i dati tra il tuo computer e la versione online è il seguente:

  1. Lavora Localmente: Inserisci tutti i tuoi dati nell'app aprendo il file index.html sul tuo computer.

  2. Esporta: Prima di aggiornare il sito, clicca su "Esporta Dati" e salva il file gestione_backup.json in un posto sicuro.

  3. Aggiorna GitHub: Carica il nuovo file index.html (e gli altri file, se ci fossero) sulla tua repository di GitHub Pages.

  4. Importa sul Sito Online: Vai all'indirizzo della tua app su GitHub Pages, clicca su "Importa Dati" e seleziona il file gestione_backup.json che hai appena salvato.

Dovrai fare questo passaggio solo quando vuoi "sincronizzare" i dati che hai inserito localmente con la versione online.
